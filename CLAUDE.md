# Sunday记账 - 产品设计文档

## 项目概述

- **项目名称**: Sunday记账
- **项目类型**: Windows桌面应用程序
- **核心功能**: 记录用户每一次人民币花销，支持一级/二级分类管理
- **目标用户**: 需要简单记账的个人用户

---

## 功能需求

### 1. 记账功能

| 功能 | 描述 |
|------|------|
| 新增记录 | 记录每笔花销的金额、分类、时间（默认当前时间）、备注（可选） |
| 查看记录 | 按日期查看所有记账记录 |
| 编辑记录 | 修改已有记录 |
| 删除记录 | 删除单条记录 |

### 2. 分类管理

#### 一级分类（6个）
1. **餐饮** - 日常饮食相关
2. **交通** - 出行相关
3. **居住** - 房租、水电、物业等
4. **购物** - 日常消费购物
5. **娱乐** - 旅游、电影、游戏等
6. **其他** - 上述以外的支出

#### 二级分类明细

| 一级分类 | 二级分类 |
|----------|----------|
| 餐饮 | 早餐、午餐、晚餐、饮料、水果、零食、外卖 |
| 交通 | 公交、地铁、出租车、加油、停车、高速费 |
| 居住 | 房租、房贷、水费、电费、燃气费、物业费 |
| 购物 | 服装、日用品、化妆品、电子产品、书籍 |
| 娱乐 | 电影、KTV、旅游、游戏、健身 |
| 其他 | 医疗、教育、通讯、人情、捐赠 |

### 3. 统计功能（可选后续实现）
- 按月统计各类别支出
- 简单图表展示

## 技术方案

### 方案一：Electron + React + SQLite

| 方面 | 说明 |
|------|------|
| 描述 | 使用Electron作为桌面框架，React构建界面，SQLite存储数据 |
| 优势 | 跨平台能力强、界面美观、生态成熟、开发效率高 |
| 劣势 | 安装包较大（约80-150MB）、占用内存较高、启动较慢 |

### 方案二：Tauri + Vue/React + SQLite

| 方面 | 说明 |
|------|------|
| 描述 | 使用Tauri作为桌面框架（Rust后端），搭配Vue或React |
| 优势 | 安装包小（10-30MB）、性能好、内存占用低、安全性高 |
| 劣势 | 生态比Electron小、某些系统API支持较新、学习曲线稍陡 |

### 方案三：C# WinForms/WPF + SQLite

| 方面 | 说明 |
|------|------|
| 描述 | 使用微软官方技术栈，WinForms或WPF构建界面 |
| 优势 | 原生Windows体验、性能好、与系统集成度高、安装包小 |
| 劣势 | 仅支持Windows、界面开发效率较低、代码较繁琐 |
